# ブロックくずし 仕様書

## 概要

mia-game-center内のミニゲーム第3弾。
パドルとボールを使ってブロックを壊していくアクションゲーム。
10ステージのアーケードモード＋フリープレイモードの2モード制。

**ゲームページ**: `games/breakout/index.html`

## ゲームルール

### 基本ルール

- 画面下のパドルを左右に動かしてボールを跳ね返す
- 跳ね返したボールで画面上部のブロックを破壊する
- すべてのブロックを破壊するとステージクリア
- ボールを画面下に落とすとライフが減る（無限モードを除く）
- ライフが0になるとゲームオーバー
- パドル接触の **±0.1秒以内にスペースキー** を押すと「強打（Strike）」が発動（後述）

### ブロックのサイズ

- ブロックの **横幅は従来想定の半分**（細長く密度を上げ、絵柄表現を可能にする）
- 高さは変わらず。Canvas解像度は **960×640px** のまま据え置き
- 当たり判定はブロックサイズに連動

## モード

### プレイモード

| モード | 内容 |
|--------|------|
| アーケードモード | 10ステージを1から順番にプレイ。ライフ・アイテムは継続 |
| フリープレイモード | 1ステージ単位で選んで遊べる。全10ステージ最初から選択可 |

### ライフモード

| モード | 内容 |
|--------|------|
| 3ライフ制 | カジュアル。ボールを3回落としたらゲームオーバー |
| 1ライフ制 | ハード。ボールを1回落としたら即終了 |
| 無限 | 練習向け。何回落としてもOK、スコアのみ記録 |

## 操作方法

| 操作 | 動作 |
|------|------|
| A / ← キー | パドルを左に移動 |
| D / → キー | パドルを右に移動 |
| スペースキー（発射前） | ボール発射 |
| スペースキー（プレイ中） | 強打（Strike）タイミング判定 |

### ボール発射

- ステージ開始時、ボールはパドルの上に乗っている状態
- スペースキー押下で上方向に発射
- ボールを落とした後の再開時も、パドル上から再発射
- プレイ中のスペースキーは「強打タイミング」判定として再利用される

### パドルの当たり方による軌道変化

- パドルの**中央付近**に当たる：ほぼ垂直に跳ね返る
- パドルの**端**に近いほど：横方向への角度が大きくなる
- 強打時にもこの軌道計算は適用される

## ブロックの種類

| 種類 | 説明 | スコア |
|------|------|--------|
| 普通のブロック | 1回当てて消える | +10点 |
| 硬いブロック | 2〜3回当てないと消えない | +30点（最終ヒット時のみ） |
| アイテムブロック | 壊すとアイテムが100%確定で落ちる | +20点 |
| 消えないブロック | 障害物。ボールが跳ね返るだけで壊れない | 0点 |

### ブロックの見た目

- **普通のブロック**: パステルカラー（赤・青・黄色・緑などをローテーション）
- **硬いブロック**: ヒビ割れ表現、HP残量で見た目が変化
- **アイテムブロック**: キラキラした特別な見た目（虹色？金色？）
- **消えないブロック**: グレー or 黒、頑丈そうな見た目

## アイテム（パワーアップ／パワーダウン）

### アイテム一覧

| アイテム | 効果 | 時間制限 |
|---------|------|----------|
| パドル拡張 | パドルが大きくなる | 10秒 |
| マルチボール | ボールが3つに増える | なし（ボールが残ってる間） |
| ボール加速 | ボールが速くなる | 10秒 |
| ボール減速 | ボールが遅くなる | 10秒 |
| 貫通弾 | ブロックを貫通する | 10秒 |
| 追加ライフ | ライフ+1 | 即時 |
| スティッキーパドル | パドルにボールがくっつき、スペースキーで再発射 | 10秒 |
| バリア | パドル下のセーフネット。ボールが1回跳ね返って消える | 壊れるまで無制限 |
| パドル縮小（デバフ） | パドルが小さくなる | 10秒 |
| ボール巨大化 | ボールが大きくなって当たりやすくなる | 10秒 |

### アイテムの出現方法

- **アイテムブロックを破壊**: 100%確定でアイテムが落下
- **通常ブロックを破壊**: 5%の確率でアイテムが落下
- **共通条件**: パドルでキャッチした時のみ発動
- **キャッチ失敗**: 画面下に落ちたアイテムは消滅、発動しない

### アイテムの見た目

- **判別不能**：すべてのアイテムは同じ見た目で落下（カプセル状）
- プレイヤーは何が出るか分からないドキドキ感あり
- パドル縮小（デバフ）も同じ見た目で混ざるため、取るかどうかの戦略性が生まれる

### アイテム重複時の挙動

- 同じアイテムを重ねて取得：効果時間がリセット
- 反対効果のアイテム（加速⇔減速、拡張⇔縮小）：後から取った方が上書き

## マルチボールの仕様

- 取得時、現在のボール1個 → 3個に増える
- 増えたボールが落ちても**ライフは減らない**
- 全部のボールが落ちたら通常通りライフ消費
- 時間制限なし、最後の1個になるまで複数ボールで遊べる

## スコアシステム

| アクション | スコア |
|-----------|--------|
| 普通のブロック破壊 | +10点 |
| 硬いブロック破壊（最終ヒット） | +30点（途中のヒットは0点） |
| アイテムブロック破壊 | +20点 |
| コンボボーナス | 連続破壊×5点（最大+50点／連続が途切れるとリセット） |
| 残ライフボーナス（ステージクリア時） | 残ライフ × 500点 |
| ステージクリアボーナス | 1000点 |
| 全クリアボーナス（アーケード10ステージ達成） | 5000点 |

### コンボの定義

- パドルでボールを跳ね返すまでに連続でブロックを破壊した数
- パドルに当たるとコンボリセット
- ボールが落ちてもコンボリセット

## ステージ構成

### 全10ステージ（テーマ別の絵柄ステージ）

各ステージはテーマに沿った **絵柄・模様** で構成。難易度はステージ順で上昇させつつ、テーマで彩る。

| ステージ | テーマ |
|---------|--------|
| 1 | ハート💕 |
| 2 | 星✨ |
| 3 | 花🌸 |
| 4 | 猫🐈 |
| 5 | 月と雲🌙 |
| 6 | リボン🎀 |
| 7 | インベーダー風👾 |
| 8 | 雪の結晶❄ |
| 9 | クラウン👑 |
| 10 | **ボス：パソコンの回路**（詳細は下記） |

#### 設計方針

- 配置は Claude Code が各テーマに沿って自由に作成
- 「壊せないブロック（X）」を装飾（輪郭・骨組み）として使用するのもOK
  - 例：ハートの輪郭を消えないブロックで描き、中身を普通ブロックで埋める
- ステージ番号が上がるほど、硬いブロック・消えないブロック・アイテムブロックの比率を上げて難易度を調整

### ボスステージ（ステージ10）詳細仕様

#### コンセプト

パソコンの基板・回路図をモチーフにした、超高難度の最終ステージ。
壊せるブロックが壊せないブロックで作られた回路パターンに組み込まれており、狭い侵入口から入り込んで攻略する必要がある。

#### グリッドサイズ

- **横18マス × 縦10マス = 180マス**
- ブロックは横半分サイズの実装系（COLS=18）に従う

#### 使用ブロック種類

| 記号 | 種類 | 役割 |
|------|------|------|
| `█` (X) | 壊せないブロック | 回路の外側・配線の外枠を構成。装飾兼障害物 |
| `▒` (N) | 壊せるブロック（普通） | 回路の中身。これを全て破壊することがクリア条件 |
| `◆` (H) | 硬いブロック | 接合点・部品を表現（複数回ヒットで破壊） |
| `○` (I) | アイテムブロック | CPUコアを表現。100%確定でアイテム落下 |
| `.` | 空白 | ボールの通り道 |

#### 回路の表現ルール

- **配線**: 「壊せないブロック → 壊せるブロック → 壊せないブロック」の3段サンドイッチ構造
  - 真ん中の壊せるブロックを壊さないと回路が完成しない演出
- **接合点**: 配線の交差点や要所に硬いブロック（◆）を配置
- **CPUコア**: アイテムブロック（○）を中央付近に配置

#### 侵入口

- **計4箇所、すべて1ブロック分の隙間**
  - 上部：列4、列9、列14（3箇所）
  - 下部：列6（1箇所）
- 狭い侵入口からしかボールが入り込めないため、攻略には角度コントロールが必須

#### ブロック数集計

- **壊せるブロック（▒/N）**: 約71個
- **硬いブロック（◆/H）**: 5個（列6行02、列13行02、列8行05、列12行07、列8行08）
- **アイテムブロック（○/I）**: 2個（列9行04、列10行06）
- **壊せないブロック（█/X）**: 残り（約96個）

#### 設計ルール

##### ①完全囲み禁止

壊せるブロック（▒・◆・○）は、**上下左右4方向のうち少なくとも1辺**が空間（.）または他の壊せるブロックに接していなければならない。
完全に壊せないブロックで囲まれてしまうとボールが到達できずクリア不能になるため、配置時は必ずチェックする。

##### ②非対称構造

- 左側：縦に長く伸びる配線パターン
- 右側：複雑な区画と短い配線
- 上下も対称ではなく、上部に配線・下部に太い回路バスが走る構造

##### ③侵入口の確保

- 上部3箇所＋下部1箇所、計4箇所
- すべて1ブロック分の隙間
- 侵入口を増やしたり広げたりしない（難易度設計の核）

##### ④横通路（行06）

- 左右端（列1・列20）にボールが通り抜けられる通路を確保
- 横通路を通ることで各区画にアクセス可能

#### クリア条件

- 通常ステージと同様、すべての壊せるブロック（▒・◆・○）を破壊
- 壊せないブロック（█）は無視（破壊不要）

#### 想定される攻略要素

- **角度コントロール**: パドルの当たり位置によるボール軌道変化を活用
- **強打システム**: 硬いブロック攻略や速攻クリアに有効
- **アイテム活用**: 貫通弾は回路の奥まで届かせるのに便利
- **マルチボール**: 複数の区画を同時攻略できる

#### 実装時の注意

- ボール挙動: 狭い通路で複数回跳ね返りが連続するため、めり込み防止の衝突判定を厳密に
- アイテム落下: 壊せないブロックに囲まれた場所でアイテム発動した場合、パドルまで届かない可能性があるため、出現位置に注意
- パフォーマンス: ブロック総数が多めのため、描画・衝突判定の最適化を推奨

## 強打システム（Strike）

### 概要

ボールがパドルに接触する瞬間（±0.1秒）にスペースキーを押すと、**強打（Strike）** が発動する。

### 判定仕様

- **判定タイミング**: ボールがパドルに触れる ±0.1秒以内のスペースキー押下
- **判定成功**: 強打発動
- **判定失敗**: 通常の跳ね返りとして処理

### 強打の効果

| 項目 | 効果 |
|------|------|
| 持続時間 | 強打成功後 **0.5秒間** |
| 速度 | 通常の **1.7倍** にアップ |
| 貫通 | 0.5秒間、ブロックを貫通する |
| スコアボーナス | **なし**（演出と気持ちよさ重視） |

### 演出

- **強打成功時**:
  - パドルが一瞬光るエフェクト
  - ボールにキラキラした軌跡エフェクト
  - 「**Strike!**」のテキスト表示（短時間）
- **強打失敗時**: 特に演出なし（通常の跳ね返り）

### キャラセリフ（追加）

- 強打成功時に `breakout.strike` セリフを発火
- 連続強打時は **3秒に1回まで** のクールタイムでテンポを崩さない
- 全キャラ（ミア／ご主人／レイン／しおり）に追加

## デザイン方針

### Canvas描画

- **解像度**: 960×640px推奨（4:3に近いアスペクト比）
- **背景**: パステルカラーのグラデーション（淡いピンク〜クリーム）
- **ボーダー**: 角丸、パステルピンクの枠

### ブロック

- 各色パステルカラー
- 角丸、ふんわりした影
- 破壊時のパーティクルエフェクト（粒子が飛び散る）

### パドル

- 横長の角丸パドル
- パステルブルー or パステルピンク
- 拡張時・縮小時はサイズが変わる

### ボール

- 白い丸（または可愛い猫の肉球柄？）
- 巨大化時はサイズが大きくなる

### アイテム

- カプセル状の落下物
- 中身が判別できない統一デザイン
- ふわっと落下するアニメーション

## 画面サイズ

- **PCメイン**（最小幅1024px推奨）
- ゲーム盤面は固定サイズ（960×640px）
